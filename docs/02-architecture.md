# FoodShare — 架构设计

## 1. 架构总览

全站基于 Cloudflare 免费层产品，无传统服务器。



## 2. 技术选型

### 2.1 前端

| 项 | 选型 | 理由 |
|-----|------|------|
| 框架 | **Astro + React** | Astro 做 SSG/SSR，React 做交互组件，最小 JS 体积 |
| 样式 | **Tailwind CSS** | 开发快，体积小，响应式简单 |
| PWA | **Vite PWA 插件** | Service Worker 自动生成 |
| 部署 | **Cloudflare Pages** | 免费，Git 集成自动部署 |

### 2.2 后端 / API

| 项 | 选型 | 理由 |
|-----|------|------|
| 运行时 | **Cloudflare Pages Functions** | 基于 Workers，与 Pages 一体化部署 |
| API 风格 | **REST** | 简单直接，小项目不需要 GraphQL |
| 路由 | **文件系统路由** | Pages Functions 原生支持 |
| 认证 | **JWT + HttpOnly Cookie** | 无状态，轻量，安全 |

### 2.3 数据存储

| 服务 | 用途 | 免费额度 |
|------|------|----------|
| **D1** | 主数据库（用户、发布、评论、点赞、关注） | 5GB 存储，500万行读/天，10万行写/天 |
| **R2** | 图片存储 | 10GB 存储，无出带宽费 |
| **KV** | 会话缓存（可选） | 10万次读/天 |

## 6. 关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 图片压缩 | 前端压缩 | 省 Workers CPU 时间，减少上传体积 |
| 图片访问 | R2 公开桶 + 自定义域名 | 无需 Worker 代理，直接 CDN 分发 |
| 分页 | Cursor-based | 比 offset 更高效，无重复问题 |
| 密码哈希 | bcrypt | 安全标准 |
| 前端架构 | Astro + React islands | 大部分静态，只在交互部分加载 JS |

## 7. 安全考虑

- 密码 bcrypt 哈希存储，永不明文
- JWT HttpOnly Cookie，防 XSS 窃取
- API 输入验证（参数类型、长度限制）
- 图片上传限制文件类型和大小
- CSRF 保护（SameSite Cookie）
