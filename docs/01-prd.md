# FoodShare — 产品需求文档 (PRD)

## 1. 项目概述

**一句话描述**：一个纯食物版的豆瓣——轻量级食物分享社区，用户可以发布食物动态，浏览朋友的分享，互相交流。

**项目背景**：个人项目，先自己用，同时分享给几十个朋友。不追求大规模用户增长，重点是体验好、简洁、零成本运维。

## 2. 目标用户

| 角色 | 描述 |
|------|------|
| 主用户 | 项目创建者自己，高频发布食物内容 |
| 朋友 | 几十人规模，浏览、点赞、评论，偶尔发布 |

## 3. 核心功能（MVP）

### 3.1 食物发布
- 发布食物动态：照片（必填）+ 文字描述 + 标签
- 支持多图上传（最多 9 张）
- 标签系统：营养健康 / 家常菜 / 烘焙 / 探店 / 食谱 等自定义标签
- 可选：地理位置、餐厅名称

### 3.2 信息流
- 首页 Feed：按时间线展示所有人的动态
- 个人主页：查看某个用户的所有发布
- 按标签筛选

### 3.3 社交互动
- 点赞
- 评论（单层即可，MVP 不做嵌套回复）
- 关注用户

### 3.4 用户系统
- 注册 / 登录（邮箱 + 密码，或 OAuth）
- 个人资料：头像、昵称、简介
- 设置页

### 3.5 PWA 支持
- 可添加到手机主屏
- 离线基础页面缓存
- 响应式设计，移动端优先

## 4. 非功能需求

| 项 | 要求 |
|-----|------|
| 成本 | 全部使用 Cloudflare 免费层，零费用运维 |
| 规模 | 支持 ~50 用户，日活 ~10-20 |
| 性能 | 页面加载 < 2s（优秀网络条件） |
| 可用性 | 依赖 Cloudflare 基础设施，无需自己维护服务器 |
| 平台 | Web 优先，PWA 支持移动端 |

## 5. MVP 不做的事情

- 私信 / 即时聊天
- 推荐算法 / 个性化 Feed
- 原生移动 App
- 第三方登录（MVP 可只做邮箱密码）
- 食谱结构化数据（步骤、用量等）
- 商业化功能（广告、付费）
- 国际化 / 多语言
- 嵌套评论 / 回复

## 6. 成功指标

| 指标 | 目标 |
|------|------|
| 能发布带图片的食物动态 | Yes |
| 朋友能注册并浏览 Feed | Yes |
| 能点赞和评论 | Yes |
| 全站零费用运行 | Yes |
| PWA 可添加到手机主屏 | Yes |
